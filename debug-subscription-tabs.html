🔍 الأسباب المحتملة

  • useEffect يراقب URL: قد يثير تغيير URL useEffect آخر
  • تداخل في الـ callbacks: onTabChange قد يستدعي setActiveTab مرة أخرى
  • تحديث الحالة المتأخر: React state updates قد تكون غير متزامنة
  • localStorage conflicts: قراءة/كتابة متضاربة في localStorage

✅ الحلول المقترحة

  1. إزالة تحديث URL: تجنب تحديث URL في setActiveTab
  2. استخدام useRef: لتتبع آخر تغيير وتجنب التداخل
  3. debounce: تأخير تنفيذ التغييرات لتجنب النقرات السريعة
  4. فصل المنطق: فصل حفظ localStorage عن تحديث الحالة
