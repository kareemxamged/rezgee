🚨 إصلاح مشكلة تضارب سجلات الحظر

حل مشكلة "المستخدم محظور بالفعل" رغم عدم ظهور ذلك في الواجهة

🔍 تحليل المشكلة

الأعراض:

  • الواجهة تظهر أن المستخدم غير محظور
  • عند محاولة الحظر تظهر رسالة "المستخدم محظور بالفعل"
  • عدم تطابق بين حالة قاعدة البيانات والواجهة

رسالة الخطأ من الكونسول:

supabase.ts:1320 ⚠️ User is already blocked messages:151 Error blocking user: المستخدم محظور بالفعل

السبب الجذري:

يوجد سجل حظر نشط في قاعدة البيانات (status = 'active') لكن الواجهة لا تعكس هذه الحالة بشكل صحيح.

✅ الحل المطبق

1. تعديل منطق الحظر:

// بدلاً من رفض الحظر عند وجود سجل نشط if (existingActiveBlock) { console.log('⚠️ Found existing active block, but allowing re-block to fix inconsistencies'); // السماح بإعادة الحظر لإصلاح التضارب }

2. معالجة السجلات الموجودة:

if (existingActiveBlock) { // تحديث السجل النشط الموجود const { data: updateResult } = await supabase .from('user_blocks') .update({ conversation_id: conversationId, block_type: 'global', status: 'active', blocked_at: currentTime, updated_at: currentTime }) .eq('id', existingActiveBlock.id); }

3. دالة تنظيف السجلات:

// دالة جديدة لتنظيف السجلات المتضاربة async cleanupBlockRecords(blockerId: string, blockedUserId: string) { // جلب جميع سجلات الحظر // الاحتفاظ بالأحدث فقط // إلغاء تفعيل السجلات القديمة }

🧪 خطوات الاختبار

  1. الخطوة 1: جرب حظر المستخدم المتأثر مرة أخرى
  2. الخطوة 2: تحقق من عدم ظهور رسالة "المستخدم محظور بالفعل"
  3. الخطوة 3: تأكد من أن الحظر يعمل بشكل صحيح
  4. الخطوة 4: جرب إلغاء الحظر ثم إعادة الحظر للتأكد من عدم تكرار المشكلة
ملاحظة مهمة: إذا استمرت المشكلة، قد تحتاج لتنظيف قاعدة البيانات يدوياً من خلال حذف أو تعديل السجلات المتضاربة في جدول user_blocks.

🔧 استعلامات قاعدة البيانات للتحقق

للتحقق من سجلات الحظر:

SELECT * FROM user_blocks WHERE blocker_id = 'f7d18de3-9102-4c40-a01a-a34f863ce319' AND blocked_user_id = '27630074-bb7d-4c84-9922-45b21e699a8c';

لإصلاح السجلات المتضاربة يدوياً:

-- إلغاء تفعيل جميع السجلات القديمة UPDATE user_blocks SET status = 'inactive', updated_at = NOW() WHERE blocker_id = 'f7d18de3-9102-4c40-a01a-a34f863ce319' AND blocked_user_id = '27630074-bb7d-4c84-9922-45b21e699a8c' AND status = 'active';
✅ النتيجة المتوقعة: بعد تطبيق هذا الإصلاح، يجب أن تعمل عملية الحظر بشكل طبيعي دون ظهور رسائل خطأ، حتى لو كانت هناك سجلات متضاربة في قاعدة البيانات.
